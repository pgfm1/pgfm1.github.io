meager run mode (mrm) [design]

vcs:n/a -- [library, scala]

Uniform handling of different run modes for Cats Effect. Here a run mode is defined as something (usually at the top) that defines how to choose, run, and evaluate some effect.

Let's say you draft a refactor of an existing feature: you might want to run the new code in the background at the same time as the old code, and then compare the results after both finish. After you are confident in the feature, you may temporarily flip this relationship to enable the new feature fully but still compare results.

Another case could be percentage-based distribution of requests, but only one-or-the-other piece of code is chosen. Similarly, you could simply alternate between two implementations. You could also run both and select the result which becomes available first.

The mrm library also provides diagnostic output which can be used (or ignored) for logging and statistics tracking for the impacted code paths.
